# Security Policy

## Reporting a vulnerability
Please **do not** open a public issue. Email security@cpmai-prep.example with
details and reproduction steps. We aim to acknowledge within 48 hours.

## Scope
- Authentication / authorization bypass
- Injection (SQL, command, prompt)
- XSS, CSRF
- Sensitive data exposure
- Insecure direct object references (IDOR)
- Cryptographic weaknesses

## Out of scope
- Denial of service via rate-limit exhaustion
- Self-XSS in dev environments
- Vulnerabilities requiring physical access
